Active Directory

2023-08-24

Introduction:⌗

Today I will be starting my first ever Active Directory Home Lab.

Requirements:⌗ 

- Virtual machine from any vendor of your choice.
- Windows Server 2019 ISO.
- Window 10 ISO.

Requirements Installation:⌗

-VMware:⌗

Installation Link, Either press the provided link or search for Vmware Player Workstation, then simply scroll down and you will see the following picture:

Choose which version you want depending on the OS you are using and press download and simply install the app.

You can add Enhanced Keyboard driver depending on preference, but it isn’t necessary.

And VMware would be installed.

-Windows Server 2019:⌗

Installation Link, Either press the provided link or search for Windows Server 2019 and choose ISO downloads:

After the installation is finished, head on to Vmware and follow the following steps.

After pressing create link you will be presented with this windows.

Choose I will install the operating system later, because if you choose ISO installation immediately you might face some installation error later on after lunching the server.

The storage can be changed depending on your Disk and PC.

Make sure to add two network adapters, one that is dedicated for the internet that is going to be running NAT, and one dedicated for the internal VMware network (Host-Only). After setting up everything, right click on the newly created virtual machine for windows server 2019 and press settings .

or you can left click on the windows machine and you can find setting also under play the virtual machine.

After you lunch the settings window, go to the CD/DVD (SATA) and choose Use ISO image file, then browse to where you downloaded your windows server ISO image and choose it.

Then after setting everything up correctly, we lunch the virtual machine. Note that once you lunch the Windows Server press enter in the machine window, because otherwise it would show a timeout error. It should show you an installation window, and that’s how you would know that it was setup correctly.

Finally setup your admin account, and you will be done with the installation.

-Windows 10:⌗

Installation Link Either press the provided link or search for Windows 10, and download Create windows 10 installation media:

After the installation is complete, lunch the installation media and follow the steps:

After the installation is completed for the ISO file, follow the same steps for the machine setup as Windows server, but make sure to choose ISO file during the initial setup.

The system configurations can be changed depending on your PC. After the setup is complete, lunch the machine and proceed with the setup

You can choose either Windows 10 Home or Windows 10 Pro. Then proceed with the installation normally and setup a random Microsoft account or proceed without one by choosing limited options.

Windows Server 2019 setup:⌗

After installing the requirements, we should setup everything to be able to start. We will start first by setting up windows server 2019 by creating a new virtual machine on your Virtual Machine, I chose VMware since it get updated more often, which is more secure and less vulnerable. Next, we should configure the network adapter setting for the internal network. To configure the network, we open control panel –> Network and Internet –> Network and Sharing Center –> Change adapter settings and you should have the following page.

I renamed the two network adapters to internal and internet so that I would be able to tell the difference between them. To know which I simply pressed right click –> status.

The one that has IPv4 connectivity is the one that is running on NAT, while the one that isn’t connected to anything is the internal internet adapter. After configuring the name, right click again on the internal adapter and choose properties, then go to the IPv4 and choose properties.

Use the following IP addresses is where we configure the IP for the network. After setting up your IP address the subnet mask should be assigned automatically according to the IP you setup and we won’t assign a default gateway, because the domain controller will serve as a default gateway, it has two NICs one on the internet and one on the inside and that’s why the internal NIC won’t have a default gateway. For the DNS server, when we install the active directory, it automatically installs the DNS and that’s why it will use itself as the DNS server. To achieve that we can either put its own IP address or the loop address 127.0.0.1.

Setting Up Active Directory on Windows Server 2019:⌗

After setting up the network setting and configuring all the basics, it is time now to setup the main even Mr. Active Directory. To setup the AD, we need Server manager which is installed by default on the windows Server.

And we press the second option, which is Add roles and features and you keep first two tabs on default by pressing next until you reach server selection tab.

Over here we want to choose the server we want to install the active directory on, in my case I only have one server, so I simply will select it and press next to install the AD on it. Then choose Active Directory Domain services, then continue with next until you reach the result tab and press install.

After the installation is done, close the installation page and you will have a warning sign on the flag located on the navigation bar, which says that you installed the AD Domain Services, but you didn’t configure it yet. Press on the flag then promote this server to a domain controller.

After you open the new tab to configure the AD, we want to choose a Add a new forest and choose a domain name then press next and setup a password for your service restore mode, then leave everything else on other tabs on default until you reach the Prerequisites Check where it will check if you have everything setup correctly, if it is all setup correctly you will be able to start the installation.

After the installation is done, it will ask you to restart your system to install and apply all changes and you simply press close.

After the setup is done you will have a new account depending on the domain name you chose, in my case it was MR\ADMINISTRATOR. Login into the account and lets setup a new user, so that we refrain from using the built in admin account. To create the new account, press windows logo –> Windows Administrative Tools –> Active Directory Users and Computers.

After lunching the Active Directory Users and Computers, we want to create a new unit, press right click on the domain you created –> New –> Organizational Unit.

Then name your new unit.

After we create the new unit we want to create a new user inside of the unit, press right click on the unit you created –> New –> User. Choose a name and password for your user, best practice in organizations is to put the first initial of the role - and the first name of the person

The options under the password are optional, I turned off user must change password because it is a lab, and password never expires because it is a lab and I don’t want to always change it in a lab.

After we have created our user, we need to configure user permissions to admin level, press right click on the user –> Properties –> Member of – > Add. And type into the object name domain admins then switch to the new created user.

Next, we need to install RAS/NAT, which is Remote Access Server/Network Address Translation to allow our windows client to be on the network privately and still connect to the internet. Same as last time with setting up the AD Domain Service, we choose Add roles and features in the server manager then leave everything on default for the first two tabs, but this time we need to add Remote Access option.

On the role service we need to add the routing feature, then we install the new features.

After the installation is done, we close the installation window. This time instead of pressing the yellow warning we need to choose tool tab –> Routing and Remote Access.

Then to configure the network to be active and able to connect to the internet, right click on DC (local) –> Configure and Enable Routing and Remote Access.

Then press next, and choose Network address translation (NAT) to enable us to connect to the internet while staying private.

Then, choose use this public interface to connect to the internet and choose the one that we named INTERNET and is connected to the internet through NAT, then press next and finish to setup everything.

You should have a similar window if everything was setup correctly.

For the next step, we want to setup the DHCP on our Domain Controller to allow our Windows 10 client to get on the internet. To setup the DHCP, we have to go back to the Add roles and features and we keep everything for the first two tabs on default, choose our server and add DHCP Server on the Server Roles, then next and install. After installation is done close the installation.

It is time now to configure the DHCP, press Tools tab –> DHCP. And you should have the following window.

Choose either IPv4 or IPv6 –> right click –> new scope.

Next, setup the scope name, for me I just named it after the range of the IP.

Then, we need to setup the range of the IP that we want to give to our clients, depending on the size of your organization you can give out more IPs, but for now we can just keep it between 100-200.

Next, we have this window, which is just asking what IPs you want to exclude or reserve, in my case I left it empty as I don’t need to exclude any.

Next window, we have the Lease Duration, which reserve an IP for a period of time for the user.

Next window, asks us if we want to configure the DHCP option, which tells the client which server to choose for DNS or Gateway, and since we want to allow the client to get on the internet, we should choose yes.

Over here we are choosing the default IP for the gateway to allow the client to connect to the internet, and I chose the Domain Controller’s IP, since I want to be the gateway, then press add. For the following windows keep everything on default and finish the setup. After the setup is complete, we want to authorize our server and refresh it so that it comes online. To do so, right click on the DC server and press authorize, then right click again and refresh

And we should see something like this.

For testing purposes we will temporarily turn on the ability to browse the internet on the DC without interruptions by security popups.

To allow access to the internet, navigate to server manager and press configure this local server –> IE Enhanced Security Configuration and turn off both available options.

Next, we need to download a PowerShell script which Josh Madakor provide us that will help us to create all users Link Reference. After the installation is finished, save the zip file somewhere easy to access and extract the files inside. Navigate to the name.txt and add a name of a user you want to create (Optional). After you setup the name of users you would like to add, navigate to the windows icon –> Windows PowerShell Folder –> PowerShell ISE (Run as administrator).

After you lunch the PowerShell, open 1_CREATE_USERS file that we downloaded by opening the folder button that has a red circle in the picture bellow and run the command in the picture bellow ‘Set-ExecutionPolicy Unrestricted’ and press yes to all. Note: in a real environment don’t allow any unknown PowerShell file to run unrestricted, as this would cause security breach in case of a malware was present.

Next, navigate to the folder we downloaded to be able to pull the names that are in the name.txt file and run the script. You can either let the script run or just stop it after certain amount of users are created.

After the script is done you can navigate back to Active Directory Active Users and Computers to check all the created users.

For the final step on DC we need to create a router on the DHCP server. To create it we just need to navigate back to the DHCP –> DC server that we created –> IPv4 –> Server Options –> right click –> Configure Options –> 003 Router –> and add the default gateway IP address. Then restart the server.

Windows 10 Client: Now all that’s left for us to do is just configure the windows 10 client by following the steps in the requirements to set the initial setup.

Then we need to setup the users, navigate to settings –> System –> About –> Rename this PC (advanced) –> Change… –> add the name of the computer and the domain –> add user name and password of one of the users we added.

After you setup the user the pc will restart and you have to enter the user name and password, then everything should be setup. You can check if the user is connected by using ipconfig and pinging mydomain.com.
