Introduction:⌗

OpenVAS (Open Vulnerability Assessment System) is an open-source vulnerability scanner used for assessing and managing security vulnerabilities in computer systems and networks. It is designed to identify potential security issues and weaknesses in software, servers, and network devices.

The Greenbone Vulnerability Management (GVM) framework is an open-source platform used for vulnerability scanning and management. It’s designed to help organizations identify and mitigate security risks within their networks and systems. The architecture of GVM comprises several key components:

1. Greenbone Security Assistant (GSA): GSA is the web interface of GVM that allows users to interact with the platform. It provides a user-friendly graphical interface for configuring scans, viewing scan results, and generating reports. GSA also facilitates user authentication and access control.

2. OpenVAS Scanner: This component is responsible for performing vulnerability scans. It uses the Network Vulnerability Tests (NVTs) database to identify vulnerabilities in target systems. The OpenVAS Scanner coordinates with the GSA for task configuration and receives instructions for conducting scans.

3. GVM Libraries (gvm-libs): These libraries provide an interface for accessing the GVM services and database. They facilitate programmatic interaction with GVM, enabling automation, integration, and customization of vulnerability management processes.

4. Greenbone Security Manager (GSM): The GSM acts as the central management system for GVM. It orchestrates various components, including the scanner, and provides functionalities for managing tasks, scheduling scans, and storing scan results. The GSM also maintains the database where scan results and configuration data are stored.

5. PostgreSQL Database: GVM relies on a PostgreSQL database to store configuration data, scan results, user information, and other relevant information. This database is crucial for maintaining a comprehensive record of security assessments.

6. OpenVAS Manager: This component is responsible for consolidating scan results from the OpenVAS Scanner. It organizes and manages the data before storing it in the PostgreSQL database. The OpenVAS Manager also handles user authentication and authorization.

7. NVTs Database: The Network Vulnerability Tests database contains a vast collection of security checks that the OpenVAS Scanner uses during vulnerability scans. These checks help identify known vulnerabilities in target systems.

8. Security Feed: The Security Feed provides regular updates to the NVTs database. It ensures that the scanner has the most current information about vulnerabilities and security issues.

9. Feed Services: These services manage the synchronization of the NVTs database with the Security Feed. They ensure that the scanner has the latest vulnerability data for effective scanning.

Scan:⌗

To start a scan using openvas:

navigate to Scans --> Tasks --> Task wizard located under the purple wand --> enter the target IP to scan.

To start a new scan under the blue star press new task.

• Name: Allows us to set the name the scan will be known as inside of OpenVAS

• Scan Targets: The targets to scan, can include Hosts, Ports, and Credentials. To create a new target you will need to follow another pop-up, this will be covered later in this task.

• Scanner: The scanner to use by default will use the OpenVAS architecture however you can set this to any scanner of your choosing in the settings menu.

• Scan Config: OpenVAS has seven different scan types you can select from and will be used based on how you aggressive or what information you want to collect from your scan.

The most important options for the scan configuration are Name, Event, Condition, and To Address.
